# DNN 学习路线图

基于当前项目（符号分类器）的扩展建议。现有代码已覆盖：架构定义、训练循环、权重分析、剪枝、量化。

---

## 一、建议新增的示例

### 1. 过拟合 + 正则化对比（最推荐）

用小数据集（100~200 条）训练，展示过拟合现象（训练准确率 100%、验证只有 70%），然后用 L1/L2 正则化、增大 Dropout 来修复。

这是 DNN 最核心的概念之一，现有代码已经有 `Dropout(0.2)` 但从未展示它为什么存在。

| 方案 | 实现方式 |
|------|---------|
| L1 正则 | `nn.L1Loss()` 叠加到总损失上 |
| L2 正则（Weight Decay） | `Adam(weight_decay=...)` |
| 增大 Dropout | `nn.Dropout(0.2)` → `nn.Dropout(0.5)` |
| 减少模型容量 | 隐藏层 64→32→1 改为 32→16→1 |

**演示对比**：

```
训练集 200 条，无正则化 → 训练 100%，验证 72%（过拟合）
训练集 200 条，L2+Dropout → 训练 96%，验证 95%（泛化良好）
```

### 2. 优化器对比

Adam vs SGD（with/without momentum）在同样任务上的收敛曲线对比。能直观理解自适应优化器的作用。

| 优化器 | 关键参数 | 特点 |
|--------|---------|------|
| SGD | lr=0.01 | 收敛慢，对学习率敏感 |
| SGD + Momentum | lr=0.01, momentum=0.9 | 加速收敛，减少震荡 |
| Adam | lr=0.001 | 自适应学习率，收敛最快 |
| RMSprop | lr=0.001 | 处理非平稳目标 |

一行 `optim.SGD` 换 `optim.Adam` 就能演示，实现成本极低。

### 3. Batch Normalization 效果

在现有网络中插入 `nn.BatchNorm1d`，对比有无 BN 时的训练速度和最终准确率。

```
原结构: Linear → ReLU → Linear → ReLU → Linear → Sigmoid
BN 结构: Linear → BN → ReLU → Linear → BN → ReLU → Linear → Sigmoid
```

BN 让收敛更快更稳，效果在更大网络里更明显。

### 4. 学习率调度

对比不同 LR 调度策略的 loss 曲线。

| 调度器 | 特点 |
|--------|------|
| Constant | 基准对照 |
| StepLR | 每隔 N 轮衰减 γ 倍 |
| CosineAnnealingLR | 余弦衰减，末期平滑 |
| ReduceLROnPlateau | 验证 loss 停滞时自动降 |

### 5. 激活函数对比

把 ReLU 换成 Sigmoid/Tanh，展示深层网络中 Sigmoid 导致的梯度消失问题。

| 激活函数 | 深层网络表现 |
|----------|-------------|
| ReLU | 收敛快，无梯度消失 |
| LeakyReLU | 对负值区域也保留梯度 |
| Tanh | 中心对称，深层仍有梯度消失 |
| Sigmoid | 深层几乎不收敛 |

**扩展**：构造一个 10 层的网络，分别用 Sigmoid 和 ReLU，对比收敛情况，直观感受梯度消失。

## 二、建议增加的"更有趣的任务"

现有任务只看符号位（一个 bit 就能解决），网络有些"大材小用"。可以加一些真正需要网络"思考"的任务：

| 任务 | 为什么有趣 | 输出 |
|------|-----------|------|
| **奇偶性判断** | 需要看最低位（LSB），所有位都要参与处理——不再是单 bit 问题 | 二分类 |
| **是否为 2 的幂** | 需要识别位模式（恰好只有一个 1），CNN 1D 视角很适用 | 二分类 |
| **绝对值比较**（\|x\| > 1000?） | 非线性决策边界，单层感知机解决不了 | 二分类 |
| **多分类**（正/负/零） | 引入 `CrossEntropyLoss` 和 softmax 输出，真正展示 DNN 的表达能力 | 三分类 |
| **回归**（预测 x/2^31） | 引入 `MSELoss`，展示回归与分类的区别 | 回归 |
| **XOR 测试** | 经典的非线性可分问题，验证网络表达能力 | 二分类 |

## 三、优先级总览

| 优先级 | 内容 | 理由 |
|--------|------|------|
| ⭐⭐⭐ | 过拟合+正则化 | DNN 最核心概念，现有代码零覆盖 |
| ⭐⭐⭐ | 优化器对比 | 一行 `optim.SGD` 换 `optim.Adam` 就能演示 |
| ⭐⭐ | 更有趣的任务 | 让模型有真正的"工作量"，增加多样性 |
| ⭐⭐ | BatchNorm | 理解归一化在深度学习中的地位 |
| ⭐ | LR 调度 | 理解训练策略选择 |
| ⭐ | 激活函数对比 | 理解梯度消失的根源 |
