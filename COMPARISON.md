## 分析：为什么模型越大剪枝越能扛？

### 不是简单的"冗余多"

表面上看是"参数多 → 冗余多 → 剪得多"，但真正的原因是**信号路径在各层的生存概率**不同。

### 信号传递模型

MSB 信号穿过网络需要 3 跳：

```
MSB(bit31) → Layer1某神经元 → Layer2某神经元 → 输出层
```

L1 非结构化剪枝按权重绝对值从小到大淘汰。一条完整路径存活的条件是：**每一跳的权重都在当前层的排名前 (1-amount) 名**。

假设各层独立剪枝，单条路径存活的概率 ≈ (1-amount)³：

| 剪枝比例 | 单路径理论存活率 |
|----------|-----------------|
| 30% | 0.7³ = 34.3% |
| 50% | 0.5³ = 12.5% |
| 90% | 0.1³ = 0.1% |
| 95% | 0.05³ = 0.01% |

### 16 位模型：路径断在并行度不够

`16→32→16→1`，Layer1 有 32 个神经元，每个有 1 条 MSB 连接：

- **50% 剪枝**：Layer1 约 16 个神经元保留 MSB → Layer2 再剪一半 → Layer3 再剪一半。虽然有 16 条并行路径兜底，但**后层（Layer2、Layer3）中 MSB 通路权重不再具有绝对排名优势**。信号分散到多路径后，单条连接的绝对值可能低于某些"噪声"权重，被误剪。累积衰减导致准确率下降。

- **90% 剪枝**：仅剩 ~3 个 Layer1 神经元有 MSB 连接，深层权重排名完全崩溃，退化为随机猜测。

### 32 位模型：MSB 权重排名始终碾压

`32→64→32→1`，Layer1 有 64 个神经元，关键差异在于：

训练后 MSB 权重天然比其他位大 **5~7 倍**。L1 剪枝按绝对值排序淘汰时，**MSB 相关权重在所有 3 层的排名中都处于最前列**。

- **95% 剪枝**：只保留绝对值前 5% 的权重。表面看单路径存活率仅 0.01%，但实际上 MSB 权重
  在每层都是排名最高的那几个——5% 的保留配额永远优先分配给 MSB 连接。
  第一层 64 个神经元，5% 配额 = 3.2 个，恰好足够保留 MSB 通路。

- **98% 剪枝**：第一层 64 × 2% = 1.28 个神经元。此时连"保底 1 条 MSB 连接"都不够，
  因为 MSB 的 64 条连接中，第 2 名以后全部被剪，仅剩的 1 条如果恰好是 MSB→某神经元，
  还得保证该神经元到后两层的连接也排进前 2%。三层联动失败，网络崩溃。

### 核心机制

```
                16位@50%        32位@95%
Layer1 神经元      32              64
MSB 权重排名      前50%可保       始终前5%（绝对值碾压）
后层排名优势      衰减，被噪声赶超   跟随 MSB 信号，保持前列
结果              路径断裂          路径完整
```

**一句话：大模型里 MSB 权重在各层的排名优势更稳固——小模型信号在深层被噪声稀释，大模型信号始终是"全班第一"，剪谁都不会剪它。**

### 延伸：Lottery Ticket Hypothesis 视角

这是 Lottery Ticket Hypothesis 的直观验证。32 位模型 95% 剪枝后仅剩 206 个权重仍保持 100% 准确率——这个稀疏子网络在训练初期就自然形成了"关注 MSB"的权重格局，剪枝只是把不重要的连接去掉，核心子网络始终完整。
